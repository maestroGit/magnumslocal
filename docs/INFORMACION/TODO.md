# TODO del proyecto

Este archivo es un checklist ligero para tareas del proyecto. Puedes editarlo desde el editor o desde GitHub.

Formato recomendado:

- [ ] Título corto de la tarea (descripción breve)
- [ ] Otra tarea (issue/PR relacionado)

Ejemplo de uso:

- [ ] Añadir .gitignore para "app/uploads/" (evitar subir artefactos de ejecución)
- [ ] Firmar (txId|metadataHash) en servidor y devolver firma en /lotes
- [ ] Añadir pruebas de integración para flujo tx → lote → qr-with-proof

Guía rápida de flujos alternativos:

- GitHub Issues: crea issues para tareas y enlaza PRs con `#<issue-number>`.
- Projects/Boards: usa la pestaña Projects en GitHub para un tablero Kanban.
- Pull Requests: cada PR debe vincular el issue o checklist que resuelve.

Sugerencia para mantener el TODO útil:

- Mantén items breves y únicos.
- Añade etiquetas entre paréntesis como (bug), (feat), (ops).
- Mueve tareas completadas a un archivo `HISTORY.md` si quieres historial.

Automatización / Integración con CI/CD (opcional):

- Puedes añadir una acción de GitHub que verifique si `TODO.md` tiene tareas abiertas y falle el build si quedan tareas críticas.

# - Transacción a tu propia dirección:
Técnicamente no hay restricción: la red solo ve una entrada (input) y una salida (output).
Autoenvío permitido siguiendo la lógica de BTC: Puedes enviar BTC desde tu wallet a la misma dirección o a otra dirección que también controles. 
• 	Uso práctico de enviarte BTC a ti mismo:
• 	Consolidar UTXOs (fragmentos de saldo) en una sola salida.
• 	“Reorganizar” fondos dentro de tu wallet.
• 	Probar la red o verificar que tu wallet funciona correctamente.
• 	Comisiones inevitables: En la red Bitcoin, cada transacción debe incluir una comisión para los mineros. Aunque el envío sea “a ti mismo”, la transacción ocupa espacio en el bloque y debe competir con otras. Por eso siempre se paga un fee
