# **I ¿Qué es BlocksWine (BW)?** 

Es una red descentralizada de custodia de vino premium en botellas de formato Magnum, basada en blockchain para convertir cada botella en un activo digital. Y una estrategía de tesorería en bitcoin.
- “Plataforma cooperativa de vanguardia para bodegas”
- “Sistema distribuido de gestión y custodia de botellas de vino premium en formato magnum para amantes del vino ”
- “Infraestructura blockchain para la transformación del sector bodeguero”
- “Red soberana de bodegas con gobernanza digital”

Cada botella queda registrada criptográficamente en la blockchain, generando un certificado digital inmutable que garantiza origen, autenticidad y trazabilidad completa.

Convierte cada botella en su propio "certificado digital". Es como si cada QR fuera un "certificado de proiedad" de la botella que cualquiera puede verificar.¡Perfecto para el mercado de vinos premium!

# **II. ¿Qué problemas resuelve BW?**

## 1. Aportar soluciones a los desafíos del sector vinícola
- Cambio generacional: Menos consumo (cantidad por individuo), los jóvenes prefieren cerveza sobre vino.
- Necesidad de modernización sin perder autenticidad.
- Demanda de trazabilidad en mercados globalizados para poner en valor la autenticidad frente a las falsificaciones.
- Las bodegas obtienen visibilidad continua de marca y acceso a nuevos clientes, al custodiar vino premium que sigue generando valor en el mercado secundario. Cada botella tokenizada actúa como embajador digital de la bodega, reforzando su reputación, autenticidad y presencia global.
- Wine Lovers vector de crecimiento y catalizador para invertir la tendencia

## 2. Limitaciones de espacio
- Las viviendas urbanas son pequeñas.
- Una vinacoteca ocupa espacio físico considerable, lo que complica almacenar botellas en cantidad.
- "Nómadas modernos", profesionales urbanos, amantes del vino que valoran la experiencia sobre la posesión física, gente que viaja y se muda con frecuencia.La vida ya no es estática. Se mueven, cambias, viajas. Y tu pasión por el vino de alta gama se convierte en una carga pesada. Imposible de trasladar, difícil de mantener."

## 3. Democratizar la tenencia de vino de alta gama
- Facilitar el acceso del gran público a vinos de alta gama, mejorando la calidad y reduciendo los costes asociados.
- Adaptarse a estilos de vida modernos y móviles, donde viajar con frecuencia y cambiar de residencia dificulta mantener y trasladar una vinacoteca o colección personal.

## 4. Conservación adecuada
- El vino requiere condiciones específicas de temperatura, humedad y oscuridad.
- Bien conservado, el vino mantiene su calidad y su evolución positiva.
- Una vinacoteca implica inversión en equipos, consumo energético y mantenimiento.

## 5. Confianza y transparencia en el producto y el vendedor
- Demostrar cómo se ha conservado el vino, es fundamental para generar confianza en la calidad del vino.
- La reputación influye en la capacidad de venta.

## 6. Riesgo de deterioro
- Existe riesgo de pérdida total si se estropea por mala conservación.

## 7. Mejora de la liquidez para transacciones de botellas
- Vender vino no siempre es rápido ni sencillo.
- Es necesario reducir costes y riesgos al mover botellas de alta gama.

## 8. Costes ocultos
- Seguros para proteger el vino premium.
- Costes de transporte, certificación y comisiones de venta.

## 9. Regulación y fiscalidad
- Algunas jurisdicciones tienen impuestos o restricciones sobre la compraventa de alcohol.
- La inversión en vino puede tener implicaciones fiscales si se considera actividad comercial.

## 10. Búsqueda de nuevos canales de financiación
- Exploración de alternativas como Bitcoin en tesorería.

## 📊 Métricas clave del modelo

Estas métricas permiten evaluar la eficiencia financiera y el potencial de escalabilidad del sistema:

| Métrica | Definición | Fórmula |
|--------|------------|---------|
| **CAC (Costo de Adquisición de Cliente)** | Inversión media necesaria para captar un nuevo cliente. | `CAC = Gasto total en adquisición / Número de nuevos clientes` |
| **ARPU (Ingreso Promedio por Usuario)** | Ingreso medio generado por cada cliente en un periodo determinado. | `ARPU = Ingresos totales / Número de clientes activos` |
| **LTV (Valor de Vida del Cliente)** | Ingreso total proyectado que un cliente generará durante su relación con el sistema. | `LTV = ARPU × Duración media de la relación (en meses)` |

> Estas métricas pueden ajustarse por vertical (trazabilidad, branding, certificación) y por tipo de cliente (bodega, distribuidor, consumidor final). A medida que avances, puedes incorporar valores reales o proyectados para cada una.

## �️ **Arquitectura Blockchain: Diseño Híbrido Unificado**

### 🎯 **Decisión Arquitectural Fundamental**

**MagnumMaster** implementa una **blockchain híbrida unificada** que maneja tanto transacciones monetarias como registros de lotes en una sola cadena, optimizando eficiencia y garantizando interoperabilidad.

### 🔧 **Tipos de Transacciones Diferenciados**

#### 💰 **Transacciones Monetarias** (`type: 'monetary'`)
```javascript
// Estructura tradicional para pagos y transferencias
{
  id: "tx_001",
  type: "monetary",
  lote: null,
  inputs: [utxo_referencias],
  outputs: [
    { address: "wallet_destinatario", amount: 25.99 },
    { address: "wallet_cambio", amount: 14.01 }
  ]
}
```

#### 🍷 **Transacciones de Lote** (`type: 'lote'`)
```javascript
// Estructura especializada para trazabilidad vinícola
{
  id: "tx_002", 
  type: "lote",
  lote: {
    loteId: "MAGNUM_2023_001",
    nombreProducto: "Vino Tinto Premium",
    bodega: "Bodegas del Duero",
    año: 2019,
    hash: "sha256_del_json_completo"
  },
  inputs: [], // Sin inputs monetarios
  outputs: [
    { address: "certificado_autenticidad", data: "lote_completo_json" }
  ]
}
```

#### 🔄 **Transacciones Híbridas** (`type: 'hybrid'`)
```javascript
// Combinan pago y registro de lote simultáneamente
{
  id: "tx_003",
  type: "hybrid", 
  lote: { /* datos del lote */ },
  inputs: [utxo_pago],
  outputs: [
    { address: "bodega", amount: 120.00 },      // Pago por botella
    { address: "plataforma", amount: 2.00 },    // Comisión MagnumMaster
    { address: "certificado", data: "lote" }    // Registro trazabilidad
  ]
}
```

### ⚡ **Ventajas del Diseño Unificado**

1. **🔗 Integridad Única**: Una sola fuente de verdad para toda la información
2. **🤝 Interoperabilidad**: Los lotes pueden vincularse directamente con pagos  
3. **⚙️ Eficiencia Operacional**: Un solo P2P network, una sola validación
4. **📊 Trazabilidad Completa**: Historial unificado de transacciones y lotes
5. **🔄 Flexibilidad**: Permite evolución hacia nuevos tipos de transacciones

### 🛡️ **Validación Diferenciada por Tipo**

```javascript
// Validador híbrido que maneja múltiples tipos
class HybridValidator {
  validateTransaction(tx) {
    switch(tx.type) {
      case 'monetary':
        return this.validateMonetaryRules(tx);
      case 'lote': 
        return this.validateLoteRules(tx);
      case 'hybrid':
        return this.validateMonetaryRules(tx) && 
               this.validateLoteRules(tx);
    }
  }
}
```
